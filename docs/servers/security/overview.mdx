SecureMCP is the trust-native server layer in this repository. It sits on top of FastMCP and adds pluggable policy engines, trust registries, marketplace discovery, compliance reporting, provenance auditing, behavioral analysis, and real-time alerting — all wired together through a unified orchestrator.

If you are looking at the broader product shape of this repo:

- `FastMCP` is the underlying MCP engine
- `SecureMCP` is the secure server layer
- `PureCipher` is the registry and publisher product layer built on top

Use this section when you want to build or attach a secure MCP server. Use the PureCipher docs when you want the trusted registry or publisher workflow.

## Architecture

SecureMCP is organized into composable layers that can be used independently or together:

**Trust & Identity** — `TrustRegistry` maintains trust scores for every tool, combining certification status, behavioral reputation, and age into a normalized 0–1 score. `TrustFederation` extends trust across peer instances with weighted score aggregation.

**Policy & Contracts** — `PolicyEngine` evaluates access decisions through pluggable providers. `ContextBroker` negotiates capability contracts between tools and callers. Both emit events for audit trails.

**Provenance** — `ProvenanceLedger` records every tool call, resource access, and policy decision into a hash-linked, tamper-evident chain. Records include actor IDs, resource IDs, and timestamps.

**Compliance** — `ComplianceReporter` runs automated checks against configurable frameworks and generates reports with findings, severity classifications, and remediation guidance.

**Behavioral Analysis** — `BehavioralAnalyzer` detects drift from established baselines using sliding-window detectors. `EscalationEngine` triggers actions (alerts, blocks, reviews) when thresholds are crossed.

**Consent** — `ConsentGraph` tracks who granted what permissions to whom, with delegation chains and temporal scoping.

**Marketplace** — `ToolMarketplace` provides publishing, discovery, reviews, and install tracking for MCP tools. `MarketplaceDataBridge` exports marketplace state as React-compatible JSON.

**Alerts** — `SecurityEventBus` provides pub/sub with severity-based routing and composable filters. All components emit events to the bus.

**Dashboard** — `SecurityDashboard` aggregates health from all components into snapshots. `DashboardDataBridge` converts snapshots to React-compatible JSON.

**HTTP API** — `SecurityAPI` exposes all the above as REST endpoints, mountable on any FastMCP server via `mount_security_routes()`.

## Quick Start

If you're building a SecureMCP-first server, use `SecureMCP`. It wraps `FastMCP` and attaches the security layer through the same public hooks exposed by `attach_security()`, so the integration stays additive while giving SecureMCP its own server surface.

```python
from fastmcp.server.security.gateway.tool_marketplace import ToolCategory
from securemcp import SecureMCP
from securemcp.config import (
    AlertConfig,
    ComplianceConfig,
    CRLConfig,
    FederationConfig,
    ProvenanceConfig,
    RegistryConfig,
    SecurityConfig,
    ToolMarketplaceConfig,
)

# Define what you need — omit any config you don't
config = SecurityConfig(
    alerts=AlertConfig(),
    provenance=ProvenanceConfig(),
    registry=RegistryConfig(),
    tool_marketplace=ToolMarketplaceConfig(),
    federation=FederationConfig(),
    crl_config=CRLConfig(),
    compliance=ComplianceConfig(),
)

server = SecureMCP(
    "my-secure-server",
    security=config,
    mount_security_api=True,
)
ctx = server.security_context
assert ctx is not None

# Register and publish a tool
ctx.registry.register("my-tool", tool_version="1.0.0", author="acme")
ctx.tool_marketplace.publish(
    "my-tool",
    display_name="My Tool",
    description="Does useful things.",
    version="1.0.0",
    author="acme",
    categories={ToolCategory.UTILITY},
)

server.run(transport="streamable-http")
```

After starting, visit `http://localhost:8000/security/health` to verify the security layer is active.

If you're attaching SecureMCP to an existing `FastMCP` server instead, use `attach_security(server, config)` and `mount_security_routes(server)`. `SecureMCP` is the higher-level facade; the helper path remains the low-level integration surface.

For local command-line workflows, this repo exposes both `fastmcp` and `securemcp` console entrypoints.

- Use `fastmcp` for upstream FastMCP examples and project workflows.
- Use `securemcp` for SecureMCP examples and SecureMCP-branded server workflows.
- FastMCP remains the unchanged base CLI; SecureMCP layers its own sibling command on top.

`attach_security()` reads SecureMCP-owned runtime settings from `SECUREMCP_*` variables. See [SecureMCP Settings](/servers/security/settings).

## SecurityOrchestrator

`SecurityOrchestrator.bootstrap()` is a stateless factory that reads a `SecurityConfig` and returns a `SecurityContext` with all requested components instantiated, cross-wired, and ready to use.

```python
ctx = SecurityOrchestrator.bootstrap(config, server_name="my-server")
```

The bootstrap process follows a fixed dependency order: event bus → policy → contracts → provenance → reflexive → consent → registry → CRL → federation → tool marketplace → compliance → sandbox → certification → gateway → dashboard. Components that depend on others (e.g., federation needs registry + CRL) receive their dependencies automatically.

### SecurityContext

The returned `SecurityContext` gives typed access to every component:

```python
ctx.event_bus              # SecurityEventBus
ctx.policy_engine          # PolicyEngine
ctx.broker                 # ContextBroker
ctx.provenance_ledger      # ProvenanceLedger
ctx.behavioral_analyzer    # BehavioralAnalyzer
ctx.escalation_engine      # EscalationEngine
ctx.consent_graph          # ConsentGraph
ctx.registry               # TrustRegistry
ctx.tool_marketplace       # ToolMarketplace
ctx.federation             # TrustFederation
ctx.crl                    # CertificateRevocationList
ctx.compliance_reporter    # ComplianceReporter
ctx.sandbox_runner         # SandboxedRunner
ctx.certification_pipeline # CertificationPipeline
ctx.dashboard              # SecurityDashboard
ctx.middleware             # list — ordered middleware instances
ctx.gateway_tools          # dict — MCP tool callables from gateway
```

Every field is `None` unless its corresponding config section was provided. The dashboard is always created (even with an empty config) so it can report health for whatever components exist.

## SecurityConfig

`SecurityConfig` is a dataclass that controls which components are created. Each sub-config has a factory method that the orchestrator calls during bootstrap.

| Sub-config | Component created | Key parameters |
|-----------|-------------------|----------------|
| `AlertConfig` | `SecurityEventBus` | `propagate_to_components` (inject bus into all layers) |
| `PolicyConfig` | `PolicyEngine` | `providers` (list of policy providers) |
| `ContractConfig` | `ContextBroker` | `require_for_list`, `crypto_handler` |
| `ProvenanceConfig` | `ProvenanceLedger` | `record_list_operations` |
| `ReflexiveConfig` | `BehavioralAnalyzer` + `EscalationEngine` | `window_size`, rules |
| `ConsentConfig` | `ConsentGraph` | `resource_owner`, `require_for_list` |
| `RegistryConfig` | `TrustRegistry` | Pass a pre-built `registry` instance, or let it create one |
| `ToolMarketplaceConfig` | `ToolMarketplace` | Auto-wired to registry + event bus |
| `FederationConfig` | `TrustFederation` | `federation_id`; auto-wired to registry + CRL |
| `CRLConfig` | `CertificateRevocationList` | Pass a pre-built `crl` instance |
| `ComplianceConfig` | `ComplianceReporter` | `framework` (custom `ComplianceFramework`) |
| `SandboxConfig` | `SandboxedRunner` | Auto-wired to CRL |
| `CertificationConfig` | `CertificationPipeline` | Auto-wired to marketplace |
| `GatewayConfig` | `Marketplace` + `AuditAPI` | `register_tools` |

Each config class also accepts a pre-built component instance. This is useful when you need to configure a component before bootstrap:

```python
from fastmcp.server.security.config import RegistryConfig, SecurityConfig
from fastmcp.server.security.registry.registry import TrustRegistry

my_registry = TrustRegistry()
my_registry.register("pre-existing-tool", tool_version="1.0.0")

config = SecurityConfig(
    registry=RegistryConfig(registry=my_registry),
)
ctx = SecurityOrchestrator.bootstrap(config)
# ctx.registry is my_registry with "pre-existing-tool" already registered
```

## HTTP API Endpoints

All endpoints are mounted under `/security` by default (configurable via the `prefix` parameter).

| Endpoint | Description |
|----------|-------------|
| `GET /security/health` | Component health check |
| `GET /security/dashboard` | Full dashboard JSON for the React UI |
| `GET /security/marketplace` | Marketplace listings with search/filter |
| `GET /security/marketplace/{id}` | Single listing detail view |
| `GET /security/trust` | Trust registry overview with all tool scores |
| `GET /security/trust/{tool}` | Trust score breakdown for a specific tool |
| `GET /security/compliance` | Compliance report (supports `?type=full|summary|executive`) |
| `GET /security/federation` | Federation peer status |
| `GET /security/revocations` | Certificate revocation list |
| `GET /security/revocations/{tool}` | Check if a specific tool is revoked |
| `GET /security/provenance` | Provenance records (supports `?resource=&actor=&limit=`) |

## Component Independence

Each component is optional. If you only need trust scoring, create just a `TrustRegistry`. If you only need provenance, create just a `ProvenanceLedger`. The `SecurityAPI` gracefully returns `503` responses for unconfigured components.

```python
from fastmcp.server.security.registry.registry import TrustRegistry

registry = TrustRegistry()
registry.register("my-tool", tool_version="1.0.0")
score = registry.get_trust_score("my-tool")
print(f"Trust: {score.overall:.2f}")
```

You can also use the orchestrator with a minimal config — only the components you configure will be created:

```python
config = SecurityConfig(registry=RegistryConfig())
ctx = SecurityOrchestrator.bootstrap(config)
# Only ctx.registry and ctx.dashboard are populated; everything else is None
```

## Event Bus

The `SecurityEventBus` provides real-time event delivery across all components. Subscribe with filters to receive only the events you care about.

```python
from fastmcp.server.security.alerts.bus import SecurityEventBus
from fastmcp.server.security.alerts.filters import SeverityFilter
from fastmcp.server.security.alerts.models import AlertSeverity

bus = SecurityEventBus()

# Only receive WARNING and above
bus.subscribe(
    "my-handler",
    handler=lambda event: print(f"Alert: {event.message}"),
    filters=[SeverityFilter(min_severity=AlertSeverity.WARNING)],
)
```

When using the orchestrator, set `AlertConfig(propagate_to_components=True)` to automatically inject the event bus into every component that supports it.

## React Dashboard

SecureMCP includes two React frontends:

- **securemcp-dashboard.jsx** — Health monitoring, trust timeline, compliance overview, component status, and activity feed
- **securemcp-marketplace.jsx** — Tool discovery with search, category filters, reviews, and install tracking

Both accept a `liveData` prop for real-time backend data and fall back to demo data when no prop is provided. Generate the `liveData` JSON from the Python backend:

```python
from fastmcp.server.security.dashboard.data_bridge import DashboardDataBridge

bridge = DashboardDataBridge(dashboard=ctx.dashboard)
live_data = bridge.export()  # dict ready to pass as liveData prop
```

The HTTP API's `/security/dashboard` endpoint returns this same JSON, so the React components can fetch it directly.
